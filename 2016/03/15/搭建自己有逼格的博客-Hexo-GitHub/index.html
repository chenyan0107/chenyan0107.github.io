配置

Hexo安装完了之后，它现在还只是一个模板性的博客，并不属于你，所以现在我们需要把它变成我们自己的博客，首先就是去找一个自己喜欢的主题吧，这会大大提高你对自己博客的兴趣，网上有大量的主题下载，在这里推荐两个网站

  • 官方主题站
  • 知乎回答

找到了主题，当然是把它配置到我们的博客中啦，找到博客根目录，把下载好的主题放到themes文件下，并且找到根目录的_config.yml文件,在_config.yml文件中找到themes: xxxxx，修改成themes: <你找的主题文件夹名>（注意冒号后面必须有空格，否则会报错的），现在就可以重新部署一下你的Hexo

快去http://localhost:4000看看是不是已经变成你自己的主题了（某些主题需要在主题文件夹下的_config.yml文件中配置别的信息），先别太激动啦，还有一些基础信息需要配置，这里就贴一下_config.yml文件下的所有配置吧

配置 描述
title 网站标题
subtitle 网站副标题
description 网站描述
author 您的名字
language 网站使用的语言
timezone 网站时区。Hexo 默认使用您电脑的时区。时区列表。比如说：America/New_York, Japan, 和 UTC 。

更多配置请参考官方文档

开始GitHub

好了，配置好了所有的配置，相信你已经在本地有了自己的博客了，然而却还没有把它放到网上，不放网上怎么能拿他去给你的小伙伴炫耀炫耀呢，接下来就是把Hexo部署到GitHub上面去

注册GitHub账号

首先，你得有一个GitHub账号，这个就不用说了

创建仓库

登录了GitHub后，你需要创建一个仓库去存放你的博客，仓库名可得取好了哦，这可是以后访问你博客的地址（例如:xxxxxx.github.io），创建好仓库后，点击Settings->GitHubPages->Launch automatic page generator,继续点击Continue to layouts,随便选一个主题，点击Publish，到这里就完成了GitHub的基本设置

部署上GitHub

创建好了仓库以后，你需要做的就是把博客部署上你的GitHub仓库，这之前你可能先需要使用git为你的github添加ssh key，网上办法很多，你可以自行搜索一下，总之需要让你的本地机器能通过ssh访问你的github，在git-bash.exe中

如果出现 Hi <你的账户名字>! You’ve successfully authenticated, but GitHub does not provide shell access. 字样，那么现在你已经离成功只差一步了

写在最后
